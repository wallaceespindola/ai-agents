# Next.js Setup Skill

## When to Use This Skill

- Creating new Next.js projects with App Router
- Setting up API routes and middleware
- Configuring authentication and authorization
- Implementing database connections
- Setting up image optimization
- Configuring deployment environments
- Establishing project conventions and structure
- Migrating from Pages Router to App Router

## Quick Start

```
/nextjs-setup <project_description_and_requirements>
```

**Example**:
```
/nextjs-setup E-commerce dashboard with product management and authentication
```

## How It Works

The skill generates production-ready Next.js projects:

### 1. Project Structure
- **app/**: App Router with file-based routing
- **lib/**: Utilities and helpers
- **components/**: Reusable UI components
- **types/**: TypeScript type definitions
- **styles/**: Global and module CSS
- **public/**: Static assets

### 2. Next.js Features
- **App Router**: File-based routing system
- **Server Components**: Server-side rendering by default
- **API Routes**: Built-in API endpoints
- **Middleware**: Request/response processing
- **Image Optimization**: next/image component
- **Font Optimization**: next/font
- **Dynamic Import**: Code splitting

### 3. Authentication Setup
- **NextAuth.js**: Complete auth solution
- **JWT Tokens**: Stateless authentication
- **Session Management**: Client and server sessions
- **Authorization**: Role-based access control
- **Protected Routes**: Route middleware protection

### 4. Database Integration
- **Prisma ORM**: Type-safe database access
- **Connection Pooling**: PgBouncer or native
- **Migrations**: Version-controlled schema changes
- **Seeding**: Database population
- **Query Optimization**: Efficient data fetching

### 5. Performance Optimization
- **Image Optimization**: Automatic format conversion
- **Font Loading**: Reduced Cumulative Layout Shift
- **Code Splitting**: Per-route JavaScript bundles
- **Caching**: ISR and revalidation
- **Compression**: Automatic minification

### 6. Styling Options
- **Tailwind CSS**: Utility-first CSS framework
- **CSS Modules**: Scoped component styles
- **Styled Components**: CSS-in-JS library
- **PostCSS**: CSS transformation

### 7. Deployment
- **Vercel**: Optimized Next.js hosting
- **Docker**: Container deployment
- **Self-hosted**: Node.js server setup
- **Environment Management**: Multi-environment config

### Example 2: API Routes

```typescript
// app/api/users/route.ts
import { NextRequest, NextResponse } from 'next/server';
import { db } from '@/lib/db';
import { authenticate } from '@/lib/auth';

// GET /api/users
export async function GET(request: NextRequest) {
  try {
    const user = await authenticate(request);
    if (!user) {
      return NextResponse.json(
        { error: 'Unauthorized' },
        { status: 401 }
      );
    }

    const users = await db.user.findMany();
    return NextResponse.json(users);
  } catch (error) {
    return NextResponse.json(
      { error: 'Internal Server Error' },
      { status: 500 }
    );
  }
}

// POST /api/users
export async function POST(request: NextRequest) {
  try {
    const user = await authenticate(request);
    if (!user?.isAdmin) {
      return NextResponse.json(
        { error: 'Forbidden' },
        { status: 403 }
      );
    }

    const body = await request.json();
    const newUser = await db.user.create({
      data: body
    });

    return NextResponse.json(newUser, { status: 201 });
  } catch (error) {
    return NextResponse.json(
      { error: 'Invalid request' },
      { status: 400 }
    );
  }
}


// app/api/users/[id]/route.ts
import { NextRequest, NextResponse } from 'next/server';

interface RouteProps {
  params: {
    id: string;
  };
}

export async function GET(
  request: NextRequest,
  { params }: RouteProps
) {
  const user = await db.user.findUnique({
    where: { id: params.id }
  });

  if (!user) {
    return NextResponse.json(
      { error: 'Not found' },
      { status: 404 }
    );
  }

  return NextResponse.json(user);
}

export async function PUT(
  request: NextRequest,
  { params }: RouteProps
) {
  const body = await request.json();
  const user = await db.user.update({
    where: { id: params.id },
    data: body
  });

  return NextResponse.json(user);
}
```

### Example 3: Middleware and Authentication

```typescript
// middleware.ts
import { NextRequest, NextResponse } from 'next/server';
import { getToken } from 'next-auth/jwt';

export async function middleware(request: NextRequest) {
  const token = await getToken({
    req: request,
    secret: process.env.NEXTAUTH_SECRET
  });

  // Protected routes
  if (request.nextUrl.pathname.startsWith('/dashboard')) {
    if (!token) {
      return NextResponse.redirect(new URL('/auth/login', request.url));
    }
  }

  // Admin only routes
  if (request.nextUrl.pathname.startsWith('/admin')) {
    if (!token || token.role !== 'admin') {
      return NextResponse.redirect(new URL('/', request.url));
    }
  }

  return NextResponse.next();
}

export const config = {
  matcher: ['/dashboard/:path*', '/admin/:path*']
};


// app/auth/login/page.tsx
'use client';
import { signIn } from 'next-auth/react';
import { useRouter } from 'next/navigation';
import { useState } from 'react';

export default function LoginPage() {
  const router = useRouter();
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    const result = await signIn('credentials', {
      email,
      password,
      redirect: false
    });

    if (result?.ok) {
      router.push('/dashboard');
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="email"
        value={email}
        onChange={e => setEmail(e.target.value)}
      />
      <input
        type="password"
        value={password}
        onChange={e => setPassword(e.target.value)}
      />
      <button type="submit">Login</button>
    </form>
  );
}
```

// app/products/[slug]/page.tsx
import { notFound } from 'next/navigation';
import { fetchProduct } from '@/lib/products';

interface ProductPageProps {
  params: {
    slug: string;
  };
}

// Generate static pages for all products
export async function generateStaticParams() {
  const products = await fetchProducts();
  return products.map(product => ({
    slug: product.slug
  }));
}

export default async function ProductPage({ params }: ProductPageProps) {
  const product = await fetchProduct(params.slug);

  if (!product) {
    notFound();
  }

  return (
    <div>
      <h1>{product.name}</h1>
      <p>{product.description}</p>
      <p>${product.price}</p>
    </div>
  );
}

## Best Practices

### 1. Component Organization
```
app/
├── (auth)/                # Route group
│   ├── login/
│   └── register/
├── dashboard/
│   ├── layout.tsx
│   ├── page.tsx
│   └── [id]/
├── api/
│   ├── auth/
│   └── products/
└── layout.tsx
```

### 2. Data Fetching Strategy
- Use Server Components by default
- Use 'use client' only when needed (interactivity)
- Fetch data at the component level
- Use revalidation for static content

### 3. Performance Optimization
- Use next/image for all images
- Implement ISR for frequently updated content
- Optimize fonts with next/font
- Use dynamic imports for large components

### 4. Error Handling
```typescript
// app/error.tsx
'use client';

export default function Error({
  error,

    </div>

```
